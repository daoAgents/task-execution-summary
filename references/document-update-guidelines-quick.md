## 版本号命名规则

| 格式 | 示例 | 用途 |
|------|------|------|
| 标准版 | v2.0、v2.1 | 完整参考文档 |
| Quick版 | v2.0-quick | 速查文档 |

### 版本号变更规则

| 变更类型 | 版本变化 | 示例 |
|----------|----------|------|
| 功能新增/修正 | 次版本号+1 | v2.0 → v2.1 |
| 破坏性变更 | 主版本号+1 | v2.5 → v3.0 |

## 文档更新流程 Checklist

### 小修流程（错别字、格式调整）

1. 直接修改目标文档
2. 更新文档底部变更记录
3. 更新 SKILL.md 版本对照表
4. 执行简化一致性检查

### 大改流程（功能变更、结构调整）

```
变更申请 → 影响评估 → 审核 → 实施 → 验证 → 发布
```

## 版本漂移防控要点

### 定义

版本漂移指文档间出现以下不一致：
- 功能描述与实际实现不符
- 跨文档引用的版本号不一致
- Quick版与完整版内容不同步
- SKILL.md Token数与实际文件不符

### 防控规则

| 规则 | 要求 |
|------|------|
| 功能变更同步 | API/错误码/流程/模板变更须同步更新 Quick+完整版 |
| Quick版同步 | 完整版更新时评估 Quick版是否需要同步 |
| SKILL.md准确 | 版本号、Token预算、功能描述必须与实际一致 |
| 定期校准 | Token数每月校准，错误码每次变更校准 |

## 一致性检查速查表

### 检查项

| 检查类型 | 检查内容 |
|----------|----------|
| 跨文档引用 | 链接可跳转、版本号有效、术语一致 |
| 版本号 | SKILL.md对照表与实际一致、Quick/完整版主版本一致 |
| Token数据 | SKILL.md预算与实际文件大小匹配 |
| 错误码 | SKILL.md数量与 error-codes.md 一致 |

### 检查频率

| 类型 | 触发条件 | 执行人 |
|------|----------|--------|
| 即时检查 | 每次文档变更后 | 变更实施者 |
| 月度检查 | 每月最后一个工作日 | 维护负责人 |
| 季度检查 | 每季度末 | 架构委员会 |

## 变更日志格式速查

### 单行格式（推荐）

```
*变更记录：v2.0→v2.1 - 变更描述（影响范围）*
```

### 详细格式（重大变更）

```markdown
### vX.Y (YYYY-MM-DD) - 版本标题

**发布日期**: YYYY年M月D日
**版本类型**: Major/Minor Release

**新增功能**：
- 功能点1

**修复问题**：
- 问题1
```

## 版本漂移严重程度分级

| 级别 | 定义 | 响应时间 |
|------|------|----------|
| Critical | 功能描述错误、版本号矛盾 | 24小时内 |
| Medium | Token数据偏差、链接失效 | 72小时内 |
| Minor | 格式不一致、描述不精确 | 下次定期审查 |
