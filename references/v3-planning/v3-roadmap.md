## 2. 模块 B1: 自定义模板支持

### 2.1 功能概述与价值主张

**核心价值**：允许用户创建、管理和应用个性化报告生成模板，突破固定格式限制。

| 维度 | 当前痛点 | V3 解决方案 | 量化收益 |
|------|---------|------------|---------|
| **品牌一致性** | 不同团队使用不同格式 | 企业可定制符合 CI 的模板 | 品牌合规性提升至 100% |
| **场景适配** | 固定模板无法满足特殊行业需求 | 支持法律/医疗/金融等行业定制模板 | 适用场景扩展 500%+ |
| **效率提升** | 手动调整格式耗时 | 一键应用预设模板 | 格式调整时间减少 95% |

### 2.2 支持的模板格式及优先级

| 格式 | 优先级 | 支持程度 | 适用场景 |
|------|--------|---------|---------|
| **HTML** | P0 (必须) | 完整支持 | Web 展示、邮件嵌入 |
| **Markdown** | P0 (必须) | 完整支持 | Git 文档、静态站点 |
| **DOCX** | P1 (重要) | 完整支持 | Word 文档编辑 |
| **PDF** | P1 (重要) | 完整支持 | 归档、打印 |
| **PPTX** | P2 (可选) | 基础支持 | 演示文稿 |

### 2.3 核心模板变量

**基础信息类**：`task_name`, `task_type`, `execution_time`, `report_date`, `quality_score`, `author_name`, `team_name`

**内容结构类**：`chapter_N_title`, `chapter_N_content`, `summary_text`, `key_decisions`, `problems_solved`, `suggestions`

**可视化组件**：`timeline_table`, `decision_matrix`, `radar_chart_data`, `progress_chart_data`

### 2.4 模板管理功能

- **模板库**：CRUD 操作、搜索筛选、版本历史
- **模板预览**：即时渲染、多设备模拟、差异对比
- **版本控制**：Major/Minor/Patch 版本管理、回滚支持
- **分享级别**：Private → Team → Organization → Public → Marketplace
- **自动化校验**：语法检查、安全扫描、性能预估

### 2.5 API 扩展 Schema

```json
{
  "generation_options": {
    "template_variant": "custom",
    "template_id": "tpl-enterprise-standard-v2",
    "template_overrides": {
      "custom_field_project_code": "PROJ-2026-042"
    },
    "output_format": "html"
  }
}
```

## 4. 模块 B3: 外部工具集成

### 4.1 功能概述与价值主张

| 维度 | 当前痛点 | V3 解决方案 | 量化收益 |
|------|---------|------------|---------|
| **工作流断裂** | 生成后需手动搬运到各平台 | 一键同步发布 | 操作步骤减少 90% |
| **格式丢失** | 复制粘贴导致格式混乱 | 智能格式转换 | 格式还原度 > 95% |
| **版本分散** | 各平台版本不一致 | 统一源头，多端同步 | 版本一致率 100% |

### 4.2 平台支持矩阵

| 平台 | 类型 | 优先级 | 目标版本 | 上线时间 |
|------|------|--------|---------|---------|
| **Confluence** | 企业 Wiki | P0 | v3.0 | Week 8 |
| **Notion** | 知识库 | P1 | v3.1 | Week 14 |
| **飞书 (Lark)** | 协作办公 | P1 | v3.1 | Week 16 |
| **语雀 (Yuque)** | 知识管理 | P2 | v3.2 | Week 24 |

### 4.3 Confluence 集成规格

**核心子模块**：Converter (格式转换)、Publisher (页面 CRUD)、SyncManager (增量同步)、AuthModule (OAuth 2.0 + PKCE)

**发布流程**：准备验证 → 格式转换 → 创建/更新页面 → 附件处理 → 后续操作

### 4.4 安全要求

| 维度 | 要求 | 实现方式 |
|------|------|---------|
| **认证安全** | OAuth 2.0 + PKCE | 加密存储 Token，自动刷新 |
| **传输安全** | TLS 1.3 强制 | 证书锁定 |
| **数据安全** | 敏感信息脱敏 | PII 检测 + 自动遮蔽 |
| **权限最小化** | 最小必要权限 | Scope 限制 + RBAC |

## 5. 模块 B4: 历史报告管理系统

### 5.1 功能概述与价值主张

| 维度 | 当前痛点 | V3 解决方案 | 量化收益 |
|------|---------|------------|---------|
| **知识流失** | 报告生成后被遗忘 | 集中式存储 + 智能检索 | 知识复用率提升 60% |
| **经验重复** | 相同问题反复出现 | 趋势分析 + 问题模式识别 | 问题复发率降低 40% |
| **绩效追踪** | 无法量化个人成长 | 多维度指标跟踪 | 绩效评估有据可依 |

### 5.2 Report Entity 数据模型

核心字段分组：身份标识(3)、任务信息(4)、内容数据(6)、元数据(6)、时间戳(3)、发布信息(1)、状态(2)、统计(8)、版本历史(1) — **总计 25+ 字段**

### 5.3 版本控制策略

| 操作类型 | 版本变化 | 说明 |
|---------|---------|------|
| Created | v1.0.0 | 首次创建 |
| Updated | Patch +1 | 内容小改 |
| Revised | Minor +1 | 结构性修改 |
| Republished | Minor +1 | 重新发布 |

### 5.4 四大功能模块

- **版本控制**：版本列表、对比、回滚、分支
- **多维检索**：全文搜索、任务名称/类型/时间/作者/质量评分/标签/状态
- **趋势分析**：生产力指标、质量指标、问题模式、决策分析、知识积累
- **可视化**：折线图、柱状图、雷达图、饼图、热力图、词云

### 5.5 数据存储方案

**推荐架构**：PostgreSQL + Elasticsearch 双写
- 结构化查询 → PostgreSQL
- 全文检索/聚合分析 → Elasticsearch
- 版本历史 → PostgreSQL (JSONB)

## 6. 模块 B5: 团队协作功能套件

### 6.1 功能概述与价值主张

| 维度 | 当前痛点 | V3 解决方案 | 量化收益 |
|------|---------|------------|---------|
| **协作低效** | 报告需要多人参与但难以协同 | 实时协作编辑 | 协作效率提升 50%+ |
| **流程缺失** | 报告发布缺乏审批机制 | 可配置审批工作流 | 合规风险降低 70% |
| **权限混乱** | 无法精细控制访问权限 | RBAC 权限体系 | 数据安全性大幅提升 |

### 6.2 三层架构设计

```
Layer 3: 权限管理层 (RBAC)
├── 角色定义 (ADMIN/OWNER/EDITOR/REVIEWER/VIEWER)
├── 权限矩阵 (角色 × 操作)
└── 审计日志

Layer 2: 审批流程层 (Workflow Engine)
├── YAML DSL 工作流定义
├── 状态机 (Draft → Reviewing → Approved → Published)
├── 电子签名
└── 通知系统

Layer 1: 实时协作层 (Real-time Collab)
├── OT 算法 / Yjs CRDT
├── WebSocket 通信
├── 光标同步
└── 冲突解决
```

### 6.3 实时协作技术方案

**技术选型**：Yjs (CRDT) / ShareDB (OT)

**协作功能**：实时光标、在线状态、行内注释、冲突提示、@提及

### 6.4 审批工作流引擎

**状态机**：draft → reviewing → [approved → published] / [revisionRequired → reviewing] / rejected

**电子签名类型**：电子签章(低) → 电子签名(中) → 数字签名(高) → 生物识别(最高)

### 6.5 RBAC 权限系统

**五级角色层次**：ADMIN → OWNER → EDITOR → REVIEWER → VIEWER

**核心权限矩阵**：

| 操作 | ADMIN | OWNER | EDITOR | REVIEWER | VIEWER |
|------|-------|-------|--------|----------|--------|
| 查看报告 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 创建/编辑 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 删除/发布 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 审批/拒绝 | ✅ | ✅ | ❌ | ✅ | ❌ |
